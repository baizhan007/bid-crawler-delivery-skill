# 验收检查清单

交付前逐项检查，不要只看程序能不能跑。

## 范围检查

- [ ] 只包含用户要求的网站。
- [ ] 明确排除的网站没有残留目录、文件、文档描述。
- [ ] 每个网站数据分开输出。
- [ ] 每个网站按确认采集的采购相关栏目拆分 CSV。
- [ ] CSV 文件名格式为 `序号、栏目名.csv`。
- [ ] 本次无数据的栏目没有生成空 CSV。
- [ ] 日期范围写清楚，避免“最近一个月”这种相对表达。

## 目录检查

- [ ] 每个网站都有 `完整源码`。
- [ ] 每个网站都有 `部署文档`。
- [ ] 每个网站都有 `配置说明`。
- [ ] 每个网站都有 `字段映射表`。
- [ ] 每个网站都有 `采集结果`。
- [ ] 每个网站都有 `验收报告`。
- [ ] 顶层有 `交付结构说明.md`。

## 源码检查

- [ ] 每个网站源码是独立单文件 `*_爬虫.py`。
- [ ] 类和函数有中文 docstring。
- [ ] 关键业务逻辑有中文注释。
- [ ] 支持 `--days`。
- [ ] 支持 `--start-date` 和 `--end-date`。
- [ ] 支持 `--output-dir`。
- [ ] 支持 `--incremental`。
- [ ] 支持 `--state-file`。
- [ ] 支持 `--incremental-stop-seen`。
- [ ] 支持代理 `BID_SPIDER_PROXY`。
- [ ] 支持请求间隔 `BID_SPIDER_DELAY`。
- [ ] 不默认包含数据库写入代码。

## 数据检查

- [ ] CSV 编码为 `utf-8-sig`。
- [ ] CSV 包含 `webname`。
- [ ] CSV 包含 `href`。
- [ ] CSV 包含 `title`。
- [ ] CSV 包含 `publish_time`。
- [ ] CSV 包含 `msg`。
- [ ] CSV 包含 `html`。
- [ ] 标题非空率 >= 99.5%。
- [ ] 正文非空率 >= 99.5%。
- [ ] 发布时间非空率 >= 99.5%。
- [ ] `webname + href` 无重复。
- [ ] 发布时间在目标日期范围内。
- [ ] 链接是 http/https 完整链接。

## 栏目覆盖检查

- [ ] 用户要求的每个栏目都采集。
- [ ] 正式采集前已向用户反馈候选采购相关栏目，并按确认范围执行。
- [ ] “查看更多”等隐藏或二级列表已检查。
- [ ] 如果某个网站近一个月无数据，文档里说明原因，并按用户要求决定是否跳过。
- [ ] 如果源站有总数，验收报告记录 expected/fetched。
- [ ] 覆盖率 >= 98.5%。

## 文档检查

- [ ] 部署文档命令可直接复制运行。
- [ ] 配置说明写清代理和请求间隔怎么改。
- [ ] 字段映射说明能追溯每个字段来自哪里。
- [ ] 验收报告 `overall_pass` 为 true，或明确解释失败原因。
- [ ] 文档不再提旧包、旧模块或不用爬的网站。

## 清理检查

运行最终搜索：

```powershell
rg -n "python -m|bid_spider|__pycache__|\\.idea|\\.vscode|output_smoke|output_test" "交付根目录"
```

如果不是用户明确要求的内容，搜索结果应为空。

## 最终汇报

向用户汇报时只讲高信号内容：

- 交付目录路径。
- 每个网站记录数。
- 验收结果。
- 是否去掉无关文件。
- 仍需用户注意的点。
